# gbScanner 데스크톱 앱 설계 및 구현 계획

## 1. 목표

`gbScanner`는 `사람 데이터.xlsx`를 기준 데이터로 사용하고, 매주 봉투 바코드를 스캔해서 날짜별/항목별 xlsx 파일에 납부 기록을 쌓는 로컬 데스크톱 앱이다.

브라우저 서비스가 아니라 컴퓨터에서 실행하는 앱으로 만든다. 실제 USB 바코드 스캐너가 입력한 값을 받아서 사람 정보를 찾고, 사용자가 금액을 확인한 뒤 해당 날짜의 항목별 xlsx 파일에 기록한다.

핵심 흐름:

1. `사람 데이터.xlsx`의 사람 정보로 바코드를 생성한다.
2. 바코드는 사람의 고유 키와 항목을 포함한 고유값으로 만든다.
3. 앱 `시작하기` 화면에서 `시작` 버튼을 누르면 오늘 날짜 기준 작업 디렉토리를 만든다.
4. 작업 디렉토리 안에 `십일조`, `해외선교`, `기타` xlsx 파일 3개를 만든다.
5. 진행 창에서 바코드를 스캔한다.
6. 스캔한 바코드로 `사람 데이터.xlsx`에서 사람과 항목 정보를 찾는다.
7. 화면에 이름, 배우자 이름, 코드, 항목, 내기로 한 금액을 보여준다.
8. 사용자가 금액을 확인하거나 수정한 뒤 확인을 누른다.
9. 해당 항목의 날짜별 xlsx 파일에 헌금 명단 양식으로 성명과 금액을 추가한다.

## 2. 권장 앱 구조

데스크톱 앱은 `Electron` 기반을 추천한다.

권장 기술:

| 영역 | 기술 |
| --- | --- |
| 데스크톱 앱 | `Electron` |
| 화면 | `React` + `Vite` |
| 앱 내부 통신 | Electron `IPC` |
| xlsx 읽기/쓰기 | `exceljs` |
| 바코드 이미지 생성 | `bwip-js` |
| PDF 생성 | `pdfkit` |
| 날짜 처리 | `dayjs` |
| 앱 패키징 | `electron-builder` |

앱 구조:

```text
Electron 앱
  -> Renderer 화면
  -> Preload 안전 API
  -> Main Process
  -> 사람 데이터.xlsx 읽기
  -> 날짜별 작업 디렉토리 생성
  -> 항목별 xlsx 파일 생성/갱신
  -> 바코드 이미지/PDF 생성
```

중요한 원칙:

- 로컬 서버를 띄우지 않는다.
- `localhost` API를 만들지 않는다.
- xlsx 파일 읽기/쓰기는 Electron Main Process에서만 처리한다.
- 화면 쪽에서는 파일 시스템에 직접 접근하지 않고, 미리 정의한 IPC 함수만 호출한다.
- 실제 스캐너는 키보드 입력처럼 처리한다.

## 3. 사람 데이터.xlsx 구조

`사람 데이터.xlsx`에는 전체 인원의 기준 정보가 들어간다.

권장 컬럼:

| 컬럼명 | 설명 | 예시 |
| --- | --- | --- |
| 키 | 사람 고유값. 중복 금지 | `1` |
| 사람이름 | 사람 이름 | `홍길동` |
| 배우자이름 | 배우자 이름. 없으면 빈 값 | `홍홍홍` |
| 해외선교 | 해외선교 여부 | `1` 또는 `0` |
| 해외선교_금액 | 해외선교로 내기로 한 금액 | `50000` |
| 십일조 | 십일조 여부 | `1` 또는 `0` |
| 십일조_금액 | 십일조로 내기로 한 금액 | `300000` |
| 기타 | 기타 여부 | `1` 또는 `0` |
| 기타_금액 | 기타로 내기로 한 금액 | `10000` |

규칙:

- `키`는 사람을 찾는 고유값이다.
- 화면과 출력 파일에서는 `키`를 `코드`로 표시해도 된다.
- `키`는 문자열처럼 다룬다. `001` 같은 값이 들어와도 앞자리 0이 사라지면 안 된다.
- `해외선교`, `십일조`, `기타`가 `1`이면 해당 항목 바코드를 생성한다.
- `해외선교`, `십일조`, `기타`가 `0`이면 해당 항목 바코드는 생성하지 않는다.
- 금액 컬럼은 스캔 후 진행 창에서 기본 금액으로 보여준다.

예시:

| 키 | 사람이름 | 배우자이름 | 해외선교 | 해외선교_금액 | 십일조 | 십일조_금액 | 기타 | 기타_금액 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| `1` | `홍길동` | `홍홍홍` | `1` | `50000` | `0` | `` | `1` | `10000` |

이 경우 생성되는 바코드는 다음 2개다.

- 코드 `1`의 해외선교 바코드
- 코드 `1`의 기타 바코드

## 4. 날짜별 작업 디렉토리 구조

앱 `시작하기` 화면에서 `시작` 버튼을 누르면 해당 날짜의 작업 디렉토리를 만든다.

디렉토리명:

```text
YYMMDD
```

파일명:

```text
YYYYMMDD_십일조.xlsx
YYYYMMDD_해외선교.xlsx
YYYYMMDD_기타.xlsx
```

예를 들어 작업 날짜가 `2026년 1월 1일`이면 아래처럼 생성한다.

```text
260101/
  20260101_십일조.xlsx
  20260101_해외선교.xlsx
  20260101_기타.xlsx
```

권장 위치:

```text
records/
  260101/
    20260101_십일조.xlsx
    20260101_해외선교.xlsx
    20260101_기타.xlsx
```

`records/` 폴더를 두면 날짜별 결과 파일을 한곳에 모을 수 있다. 다만 사용자가 원하면 앱 설정에서 저장 위치를 바꿀 수 있게 한다.

## 5. 날짜별 항목 xlsx 구조

시작 버튼을 눌러 생성되는 3개 파일은 모두 같은 헌금 명단 양식을 사용한다.

상단 구조:

| 위치 | 내용 | 예시 |
| --- | --- | --- |
| 1행 | 항목별 헌금 명단 제목 | `십일조 헌금 명단` |
| 2행 오른쪽 | 작업 날짜 | `2026년 6월 18일` |
| 3행 | `성명`, `금액`, `성명`, `금액` 헤더 | `성명 / 금액` |
| 4행부터 | 본인 이름, 와이프 이름, 낸 금액 | `홍길동 / 홍홍홍 / 50000` |

입력 배치:

- 왼쪽 블록은 `A:B`가 성명 영역이고, `A`에는 본인 이름, `B`에는 와이프 이름을 넣는다.
- 왼쪽 블록 금액은 `C:D`를 병합한 한 칸에 넣는다.
- 오른쪽 블록 금액은 `G:H`를 병합한 한 칸에 넣는다.
- 왼쪽 블록은 위에서 아래로 25명까지 채운다.
- 왼쪽 블록이 차면 오른쪽 블록 `E:H`를 같은 방식으로 채운다.
- 한 블록에 총 50명까지 들어간다.
- 50명을 넘으면 아래에 같은 형식의 새 블록을 만든다.
- 각 블록 하단에는 왼쪽 소계, 오른쪽 소계, 합계를 넣는다.

`20260101_해외선교.xlsx` 예시:

```text
해외선교 헌금 명단
                                      2026년 1월 1일
성명           금액        성명           금액
홍길동 홍홍홍  50,000
...
소계           50,000      소계           0
계수인                     합계           50,000
```

권장 저장 방식:

- 확인 버튼을 누를 때마다 해당 항목 파일의 명단 양식에 값을 추가한다.
- 같은 사람이 같은 항목을 다시 낼 수 있으므로 기본은 누적 기록 방식으로 둔다.
- 같은 날짜 작업에서 동일한 바코드가 다시 입력되면 저장하지 않고 경고창을 띄운다.

## 6. 바코드 고유값 설계

바코드에는 사람 이름 같은 개인정보를 직접 넣지 않고, 사람 키와 항목 코드만 넣는다.

권장 형식:

```text
GBS:v1:{키}:{항목코드}:{검증값}
```

항목코드:

| 항목 | 항목코드 | 저장 파일 |
| --- | --- | --- |
| 해외선교 | `MISSION` | `YYYYMMDD_해외선교.xlsx` |
| 십일조 | `TITHE` | `YYYYMMDD_십일조.xlsx` |
| 기타 | `ETC` | `YYYYMMDD_기타.xlsx` |

예시:

```text
GBS:v1:1:MISSION:AB12CD34
GBS:v1:1:TITHE:9F8E7D6C
GBS:v1:1:ETC:1357BDF0
```

검증값 생성 방식:

- 앱 내부 비밀키를 사용해 `키 + 항목코드`로 HMAC-SHA256을 만든다.
- 앞 8~12자리만 바코드에 넣는다.
- 스캔 시 다시 검증값을 계산해서 일치하지 않으면 잘못된 바코드로 처리한다.

비밀키 관리:

- 최초 실행 시 `config/app-secret.json`을 생성한다.
- 이미 생성된 비밀키는 다시 만들지 않는다.
- 비밀키가 바뀌면 기존 바코드를 검증할 수 없으므로 `config/` 폴더도 백업해야 한다.

## 7. 바코드 생성 흐름

### 7.1 전체 생성

1. 앱에서 `사람 데이터.xlsx`를 읽는다.
2. 각 사람의 `해외선교`, `십일조`, `기타` 값을 확인한다.
3. 값이 `1`인 항목마다 바코드를 하나씩 생성한다.
4. 생성된 바코드 이미지를 PDF에 배치한다.
5. PDF 파일을 `output/barcodes/` 폴더에 저장한다.
6. 완료 후 앱에서 PDF 열기 버튼을 제공한다.

### 7.2 일부 생성

1. 사용자가 사람 코드, 즉 `키`를 입력한다.
2. 쉼표로 여러 명을 입력할 수 있게 한다. 예: `1, 3, 7`
3. 입력한 키에 해당하는 사람만 필터링한다.
4. 해당 사람들의 `1` 항목만 바코드를 생성한다.
5. 하나의 PDF로 출력한다.

### 7.3 PDF 라벨 형식

각 라벨에 표시할 정보:

- 이름
- 배우자 이름
- 항목명
- 바코드 이미지

예시:

```text
홍길동 / 홍홍홍
항목: 해외선교

[바코드 이미지]
```

## 8. 시작 및 진행 흐름

### 8.1 시작하기 화면

시작하기 화면에는 큰 `시작` 버튼을 둔다.

시작 버튼을 누르면 앱이 다음 작업을 한다.

1. 오늘 날짜를 기준으로 `YYMMDD` 디렉토리명을 계산한다.
2. `records/YYMMDD/` 디렉토리가 없으면 생성한다.
3. 아래 3개 파일이 없으면 생성한다.
   - `YYYYMMDD_십일조.xlsx`
   - `YYYYMMDD_해외선교.xlsx`
   - `YYYYMMDD_기타.xlsx`
4. 이미 파일이 있으면 기존 파일을 이어서 사용한다.
5. 진행 창을 연다.

날짜 예시:

```text
작업 날짜: 2026-01-01
디렉토리: records/260101/
파일:
  records/260101/20260101_십일조.xlsx
  records/260101/20260101_해외선교.xlsx
  records/260101/20260101_기타.xlsx
```

### 8.2 진행 창

진행 창은 실제 봉투 스캔을 처리하는 화면이다.

진행 창 기능:

- 바코드 입력칸 자동 포커스
- 스캐너 입력 감지
- 사람 정보 표시
- 항목 표시
- 내기로 한 금액 표시
- 납부 금액 수정
- 확인 버튼
- 최근 기록 목록 표시
- 현재 작업 날짜와 저장 폴더 표시

화면 표시 예시:

```text
이름: 홍길동
배우자이름: 홍홍홍
코드: 1
항목: 해외선교
내기로 한 금액: 50,000원

낸 금액: [50,000]
[확인]
```

## 9. 스캔 및 기록 저장 흐름

### 9.1 스캔 조회

1. 사용자가 진행 창에서 바코드를 스캔한다.
2. 스캐너가 바코드 텍스트와 `Enter`를 입력한다.
3. 앱이 바코드값을 파싱한다.
4. 검증값을 확인한다.
5. 바코드의 `키`로 `사람 데이터.xlsx`에서 사람을 찾는다.
6. 바코드의 항목코드로 항목명을 찾는다.
7. 해당 항목의 내기로 한 금액을 가져온다.
8. 화면에 사람 정보와 금액을 보여준다.

### 9.2 확인 저장

사용자가 확인 버튼을 누르면 다음 순서로 저장한다.

1. 금액이 숫자인지 확인한다.
2. 항목에 맞는 날짜별 xlsx 파일을 선택한다.
3. 저장 전 해당 xlsx 파일을 백업한다.
4. 파일에 새 행을 추가한다.
5. 저장 완료 후 진행 창의 최근 기록에 보여준다.
6. 다음 봉투 스캔을 위해 입력칸으로 포커스를 돌린다.

예시:

스캔한 바코드가 홍길동의 해외선교 바코드라면 화면에 다음 정보가 나온다.

```text
이름: 홍길동
배우자이름: 홍홍홍
코드: 1
항목: 해외선교
내기로 한 금액: 50,000원
```

확인 버튼을 누르면 아래 파일에 행이 추가된다.

```text
records/260101/20260101_해외선교.xlsx
```

추가되는 값:

```text
성명: 홍길동 / 홍홍홍
금액: 50,000
```

## 10. 실제 바코드 스캐너 입력 방식

대부분의 USB 바코드 스캐너는 키보드처럼 동작한다.

스캔 결과:

```text
GBS:v1:1:MISSION:AB12CD34
Enter
```

권장 스캐너 설정:

- 스캔 후 suffix는 `Enter`로 설정한다.
- prefix는 사용하지 않는다.
- Code128 바코드를 읽을 수 있게 설정한다.
- 한글은 바코드에 넣지 않고 영문/숫자/기호만 넣는다.

앱에서 처리할 예외:

- 입력칸 포커스가 사라졌을 때 다시 포커스 복구
- 너무 짧은 값 무시
- 같은 바코드가 1초 안에 중복 입력되면 중복 스캔 경고
- 잘못된 형식이면 오류 표시
- 검증값이 틀리면 저장 불가
- 사람 데이터에 없는 키면 저장 불가
- 해당 사람이 해당 항목을 하지 않는 것으로 되어 있으면 저장 전 경고

## 11. 화면 설계

### 11.1 시작하기 화면

기능:

- 오늘 날짜 표시
- 시작 버튼
- 오늘 작업 폴더 미리보기
- 사람 데이터 파일 상태 표시
- 바코드 생성 화면으로 이동
- 설정 화면으로 이동

### 11.2 바코드 생성 화면

기능:

## 14. 안정성 규칙

xlsx 파일은 일반 DB보다 충돌에 약하므로 다음 규칙을 둔다.

- 항목별 xlsx 파일 저장 전 항상 `backups/`에 백업 파일을 만든다.
- 동시에 두 번 저장되지 않도록 앱 내부 파일 쓰기 잠금을 둔다.
- 저장은 임시 파일에 먼저 쓰고 성공하면 원본 파일을 교체한다.
- 필수 컬럼이 없으면 앱 시작 시 오류를 보여준다.
- 금액은 숫자만 허용하고 음수는 막는다.
- 사람 데이터의 `키`가 중복되면 바코드 생성과 스캔 저장을 중단한다.
- 날짜별 xlsx 파일이 엑셀에서 열려 있어 저장에 실패하면 사용자에게 닫으라고 안내한다.
- 저장 성공 후 바로 파일을 다시 읽어 행이 추가됐는지 확인한다.
- 작업 날짜 폴더가 이미 있으면 새로 덮어쓰지 않고 기존 파일에 이어서 기록한다.

Electron 보안 설정:

- `contextIsolation: true`
- `nodeIntegration: false`
- Renderer에서 `fs` 직접 접근 금지
- Preload에서 허용된 IPC 함수만 노출

## 16. 현재 요구사항 기준 추천 결정

- 앱 형태: Electron 데스크톱 앱
- 바코드: `Code128`만 사용
- 스캐너 입력: USB 스캐너의 키보드 입력 방식
- 사람 기준 파일: `사람 데이터.xlsx`
- 주간 기록 방식: 시작 버튼으로 날짜별 작업 폴더 생성
- 기록 파일: 날짜별 `십일조`, `해외선교`, `기타` xlsx 3개
- 저장 방식: 확인 버튼을 누를 때 해당 항목 파일에 새 행 추가
- 금액 수정: `사람 데이터.xlsx`의 약정 금액은 변경하지 않고, 해당일 봉투의 낸 금액만 수정
- 약정 금액 변경: 앱 진행 중에는 하지 않고, 필요하면 `사람 데이터.xlsx` 원본을 별도로 수정
